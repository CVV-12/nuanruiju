1️⃣ Claude Code 是什么

Claude Code 是 Anthropic 推出的 AI 编程助手，运行在你的终端里。 它不是一个普通的聊天机器人——它可以直接读取你的项目文件、编辑代码、 运行终端命令，像一个真正的同事一样和你协作。

你可以把它理解为：一个能直接动手干活的 AI 结对编程伙伴。

它能做什么？

能力说明
🔍 阅读代码快速理解项目结构、查找函数定义、追踪引用关系
✏️ 编辑文件精确修改代码，支持单文件和跨文件重构
💻 运行命令执行 npm test、git diff 等终端命令
🔧 调试 Bug分析错误日志，定位根因并直接修复
📦 管理 Git创建分支、提交代码、生成 PR 描述
🧠 持久记忆记住你的偏好和项目约定，下次对话自动加载

2️⃣ 安装与启动

安装

# macOS / Linux
npm install -g @anthropic-ai/claude-code

# 或者用 Homebrew
brew install claude-code

启动

# 进入你的项目目录，然后运行：
claude

首次启动会让你登录 Anthropic 账号并选择订阅计划。之后就可以直接在终端里和 Claude 对话了。

建议在一个 Git 仓库的根目录下启动 Claude Code，这样它能自动感知整个项目的上下文。

3️⃣ 基本对话方式

打开 Claude Code 后，你看到的是一个交互式命令行界面。直接用自然语言描述你想做的事情即可：

# 示例对话
你：帮我在 src/utils 下创建一个 formatDate.ts 工具函数
你：这个函数有什么问题？为什么在 Safari 上报错？
你：把项目中所有的 var 改成 const/let
你：帮我解释一下 auth.ts 里的登录流程
不要只给模糊的指令。告诉 Claude 你要什么、在哪里、为什么，结果会好很多。比如「帮我在 checkout 页面加一个 loading 状态，防止用户重复提交」比「加个 loading」好得多。

4️⃣ 让 Claude 看到你的代码

Claude Code 默认可以浏览你的项目目录并读取文件。有几种方式让它理解你的代码：

方式一：直接提到文件名

你：看看 src/components/Header.tsx，帮我把这个组件改成响应式的

方式二：用 @ 引用（在 IDE 中）

如果你在 VS Code 或 JetBrains 插件中使用 Claude Code，可以用 @ 符号引用文件、文件夹或符号：

你：帮我重构 @src/services/api.ts 里的请求逻辑，加上错误重试

方式三：拖拽文件（桌面版）

在桌面版应用中，直接把文件或文件夹拖进对话窗口即可。

方式四：让 Claude 自己去探索

你：帮我了解一下这个项目的路由结构
# Claude 会自动用 ls、grep、read 等工具去探索项目
Claude 一次能看到的上下文是有限的。如果你要做一个很大的改动，建议拆成小步骤，逐步推进。

5️⃣ 让 Claude 帮你写代码

Claude Code 不仅能回答问题，还能直接修改你的文件。每次修改前，它会显示具体的 diff，让你确认。

常见场景

场景你可以这样说
新增功能「在 settings 页面加一个深色模式切换开关」
修复 Bug「用户提交表单后页面刷新了，应该阻止默认行为，帮我修一下」
重构代码「把 userService.ts 里的三个重复的 fetch 调用抽成一个公共函数」
写测试「给 calculateTotal 函数写单元测试，覆盖边界情况」
加注释「给这个复杂的 SQL 查询加上中文注释说明每一步的逻辑」
改完之后，你可以让 Claude 直接运行测试或启动项目来验证改动是否正确。比如：「改完了，帮我跑一下测试看看有没有问题」。

6️⃣ 终端命令执行

Claude Code 可以代表你执行终端命令。这在以下场景特别有用：

# 实际操作示例
你：帮我装一下 lodash 和 date-fns
→ Claude 执行：npm install lodash date-fns

你：看看我改了哪些文件
→ Claude 执行：git status && git diff

你：跑一下测试，如果有失败的帮我看原因
→ Claude 执行：npm test，然后分析失败输出

你：帮我创建一个新分支 feature/user-avatar 并切换过去
→ Claude 执行：git checkout -b feature/user-avatar
某些可能有风险的操作（如 git push --force、rm -rf、删除数据库等），Claude 会先征求你的确认。请仔细阅读确认提示再批准。

7️⃣ 斜杠命令速查

在对话中输入 / 可以触发各种内置命令：

命令作用
/help查看帮助信息
/clear清空当前对话历史，开始新话题
/compact压缩对话上下文，释放空间（保留关键信息）
/config修改配置（主题、模型等）
/review对当前分支的改动进行代码审查
/simplify自动简化当前 diff 中的代码
/security-review对当前改动进行安全审查
/init为当前项目生成 CLAUDE.md 配置文件
/run启动项目的应用并验证改动
/verify验证某个改动是否真的生效了
/loop每隔一段时间重复执行某个命令
/add-dir添加工作目录之外的文件夹到上下文
/cost查看当前会话的 token 消耗
/stats查看使用统计数据
/feedback向 Anthropic 提交产品反馈

8️⃣ 权限控制

出于安全考虑，Claude Code 在执行某些操作前需要你的确认。权限分为几个级别：

级别说明
🔴 需确认写入文件、执行命令、网络请求等——每次都需要你批准
🟡 会话内允许在当前会话中记住你的选择，同类操作不再重复询问
🟢 始终允许写入到 settings.json，以后永远不再询问（谨慎使用）

你可以在 ~/.claude/settings.json（全局）或项目中的 .claude/settings.json 里配置权限规则。

不要对 git push --force、数据库删除等危险操作设置「始终允许」。保持确认机制可以防止意外。

9️⃣ 记忆系统

Claude Code 有一个持久记忆系统，可以跨对话记住重要信息。记忆分为四种类型：

类型存储内容示例
👤 用户你的角色、偏好、技能水平「我是后端工程师，刚学 React」
💬 反馈你之前纠正过的方法和风格偏好「不要给测试 mock 数据库」
📁 项目当前项目的背景、目标、约束「下周四之前冻结合并，准备发版」
🔗 参考外部资源的位置「Bug 跟踪在 Linear 的 INGEST 项目里」

你可以直接说「记住……」，Claude 会自动分类并保存。说「我之前的偏好是什么？」可以查看已保存的记忆。

🔌 在 VS Code 中使用 Claude Code

除了终端命令行，Claude Code 还提供了 VS Code 原生扩展，让你在编辑器内直接和 Claude 协作，无需切换窗口。

安装扩展

在 VS Code 中按 Cmd+Shift+X 打开扩展市场，搜索 「Claude Code」，点击安装。或者直接运行：

code --install-extension anthropic.claude-code

三种打开方式

方式操作
🖱️ 侧边栏点击左侧活动栏的 Claude Code 图标，在侧边面板中对话
⌨️ 快捷键按 Cmd+Shift+L（Mac）或 Ctrl+Shift+L（Windows）快速打开对话面板
🔍 命令面板按 Cmd+Shift+P，输入「Claude Code: Open Chat」

核心功能：@ 文件引用

在 VS Code 扩展中，你可以用 @ 符号快速引用上下文，Claude 会直接读取对应内容：

# 引用文件
你：帮我重构 @src/utils/helper.ts，把重复逻辑抽成公共函数

# 引用文件夹（Claude 会理解整个目录结构）
你：@src/components/ 这个目录下的组件结构合理吗？帮我分析一下

# 引用当前打开的编辑器内容
你：帮我给 @active-file 加上错误处理

# 引用选中的代码
你：解释一下 @selection 这段代码的逻辑

选中代码，一键提问

在编辑器中选中一段代码，然后：

  1. 右键 → 「Claude Code: Ask About Selection」
  2. 或按快捷键 Cmd+Shift+I（Mac）/ Ctrl+Shift+I（Windows）

Claude 会自动把选中的代码作为上下文，你可以直接问「这段代码有什么问题？」或「帮我优化这个函数」。

内联 Diff 审查

当 Claude 建议修改代码时，VS Code 扩展会以 内联 diff 的方式展示改动——就像 Git 的对比视图一样。你可以：

操作说明
✅ 接受全部一键应用 Claude 建议的所有改动
❌ 拒绝全部放弃此次修改建议
📝 逐行审查在 diff 视图中逐条接受或拒绝每个变更块

终端集成

VS Code 扩展可以直接使用 VS Code 内置终端执行命令。当 Claude 需要运行 npm test 或 git status 时，会在 VS Code 终端中执行，你可以实时看到输出。

常用快捷键速查

快捷键 (Mac)快捷键 (Windows)功能
Cmd+Shift+LCtrl+Shift+L打开/关闭 Claude Code 面板
Cmd+Shift+ICtrl+Shift+I对选中的代码提问
Cmd+Shift+ACtrl+Shift+A将选中代码添加到上下文
Cmd+Shift+RCtrl+Shift+R运行当前文件或项目
在 VS Code 扩展中写 Prompt 时，多用 @ 引用具体文件和符号。上下文越精确，Claude 给出的代码和建议质量越高。
VS Code 扩展和终端 CLI 共享同一个账号和配置，但扩展有自己独立的权限设置，需要在扩展的配置页面中单独管理。

🔟 实用技巧

📋 写清楚的 Prompt

对比一下：

❌ 模糊✅ 清晰
「修一下登录」「登录接口返回 500 错误，帮我看一下 auth.controller.ts 第 42 行附近的逻辑并修复」
「加个按钮」「在 Dashboard.tsx 的顶部导航栏右侧加一个『导出 CSV』按钮，点击后调用 /api/export 接口」
「优化性能」「商品列表页滚动到 100 条之后就卡顿了，帮我分析一下渲染瓶颈并优化」

📦 把小任务拆开

一次性让 Claude 做太多事情容易出错。把一个大的需求拆成 3-5 个小步骤，逐步推进，每一步确认后再继续。

🔄 善用 /compact

当对话变得很长时，Claude 的上下文窗口可能会满。输入 /compact 可以压缩历史，释放空间继续对话。

📝 用 CLAUDE.md 记录项目约定

在项目根目录放一个 CLAUDE.md 文件，记录：

# CLAUDE.md 示例
## 项目约定
- 使用 TypeScript 严格模式
- 组件放在 src/components/ 下
- API 调用统一走 src/services/
- 测试用 Vitest，不要用 Jest
- 提交信息用中文

Claude 每次对话会自动读取这个文件，确保行为和你的项目约定一致。

🤝 把 Claude 当同事，不是搜索引擎

Claude Code 擅长的是协作式编程——讨论方案、审查代码、迭代优化。给它足够上下文，它会表现得像一个熟悉你项目的资深同事。

遇到不确定的事，直接问 Claude：「你觉得这两种方案哪个更好？为什么？」它会给出有深度的分析和建议。
